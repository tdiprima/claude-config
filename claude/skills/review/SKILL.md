---

Perform a professional code review.

Steps:

1. Identify files that changed or were provided.
2. Review architecture and structure.
3. Check for readability and maintainability issues.
4. Check for security vulnerabilities:
   - secrets in code
   - unsafe subprocess usage
   - injection risks
   - improper input validation
5. Check configuration and logging practices.

Output sections:

- Critical security issues
- Reliability problems
- Maintainability concerns
- Suggested improvements
